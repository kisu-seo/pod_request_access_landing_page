## My process

### Built with

- **Semantic HTML5 Markup** - `<main>`, `<header>`, `<section>`, `<form>` for proper document structure
- **Vanilla CSS** - Pure CSS without preprocessors or frameworks for maximum control
- **CSS Custom Properties (Variables)** - Centralized design tokens in `:root` for maintainability
- **BEM Methodology** - Block Element Modifier naming convention (`.hero__title--highlight`)
- **Component-Based CSS** - Reusable classes for buttons, forms, and layouts
- **Pixel-Perfect Design** - Precise values matching design specs (17.74px gap, 547.65px width)
- **Mobile-First Responsive** - Breakpoint-based design for 375px → 768px → 1024px+
- **Flexbox Layout** - Centering, alignment, and responsive ordering with `order` property
- **Fixed Background Images** - `background-attachment: fixed` for parallax-like effect
- **Vanilla JavaScript** - No frameworks, pure DOM manipulation with JSDoc documentation
- **Email Validation** - Regex-based client-side validation with real-time error feedback
- **Accessibility (a11y)** - `aria-live`, `aria-invalid`, `visually-hidden` class, semantic HTML
- **Form UX Patterns** - Real-time error clearing, focus states, and user-friendly messages
- **Smooth Transitions** - `transition` properties for hover effects and state changes
- **Google Fonts** - Chivo font family (Light 300, Bold 700) for typography
